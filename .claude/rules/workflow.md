# 작업 워크플로우 — Git 및 배포

> 모든 코드 수정 작업은 이 워크플로우를 따른다.
> 사장님의 기존 작업 흐름을 그대로 반영.

---

## 사장님 기존 작업 흐름

```
1. GitHub에서 최신 코드 내려받기
2. 로컬에서 수정 작업
3. 수정 끝나면 GitHub에 올리기
4. Cloud Run에 재배포
```

Claude Code는 이 흐름을 **반드시** 지킨다.

---

## 세션 시작 시 (Claude 자동 체크)

새 세션이 시작되면 Claude Code가 자동으로:

1. **CLAUDE.md 읽기** (프로젝트 규칙 로드)
2. **현재 Git 상태 확인:** git status
3. **최신 코드 pull:** git pull origin main
4. **오늘 작업 주제 파악** → 해당 규칙 파일 읽기
5. **진행상황 섹션 확인** (CLAUDE.md 하단)
6. 사장님 요청 대기

### 사장님이 매번 할 말
- "오늘은 [작업 내용] 할 거야"
- Claude가 알아서 위 체크 후 시작

## 작업 중 규칙

### 파일 수정 전 반드시 확인
1. **무엇을 왜 수정하는지** 사장님에게 먼저 설명
2. **사장님 확인(y/n)** 받고 수정 시작
3. **한 번에 한 파일씩** 수정 (큰 변경은 분할)
4. 수정 후 **diff 요약** (어디가 어떻게 바뀌었는지)
5. 사장님 확인 후 다음 파일

### 코드 수정 원칙
- **index.js 건드리지 말 것** → 새 파일/모듈로 분리
- **기존 함수 시그니처 유지** (V2 프론트 호환성)
- **주석은 한국어로** (사장님이 읽을 수 있게)
- **console.log 남발 금지**

## 작업 완료 시 (Git Commit)

### Claude가 해야 할 것

1. **변경사항 요약 보고:**
   - 어떤 파일이 추가/수정/삭제됐는지
   - 각 변경의 목적
   
2. **git status 확인:**
```
   git status
```

3. **사장님에게 커밋 메시지 제안:**
   - 제안 예시: "feat: 직업패 라우터 추가 (JIKUP_CARD)"
   - 사장님 확인 후 수정/승인

4. **git add + commit:**
```
   git add .
   git commit -m "[제안된 메시지]"
```

5. **⚠️ push는 사장님 명시적 허락 후에만:**
```
   git push origin main
```

### ❌ 절대 자동으로 하지 않는 것
- **git push** (사장님 허락 필수)
- **git reset --hard** (복구 불가)
- **git force push** (절대 금지)
- **브랜치 삭제** (사장님 확인 필수)

## 커밋 메시지 규칙

### 형식
```
[타입]: [간결한 설명]

[상세 설명 - 선택]
```

### 타입 종류
| 타입 | 용도 | 예시 |
|---|---|---|
| `feat` | 새 기능 추가 | `feat: 직업패 라우터 신설` |
| `fix` | 버그 수정 | `fix: Toss 결제 콜백 URL 오류 수정` |
| `refactor` | 코드 구조 개선 | `refactor: scoreEngine 함수 분리` |
| `docs` | 문서 수정 | `docs: CLAUDE.md 진행상황 업데이트` |
| `style` | 스타일 (CSS) 변경 | `style: 직업처방 강철블루 컬러 적용` |
| `content` | 텍스트 데이터 변경 | `content: 누수처방 Ch.3 문구 수정` |

### 예시 커밋 메시지
- `feat: 직업패 무료 카드 라우터 추가`
- `content: 재물풀이 Ch.6 직업 궁합 텍스트 전면 개편`
- `fix: calcJobPaeKey 월급 체질 판정 조건 오류`
- `refactor: 누수처방 데이터 구조 분리`

## Cloud Run 재배포

push만으로는 실서버에 반영 안 됨. **재배포**까지 해야 완료.

### 재배포 명령어 (사장님이 직접)
```
gcloud run deploy hw-server-v2 \
  --source . \
  --region asia-northeast3 \
  --allow-unauthenticated
```

### Claude가 해야 할 것
- 재배포 필요한 시점에 **알려주기만** 하기
- 실제 실행은 사장님이
- 재배포 후 사장님이 **결과 확인** (로컬 URL 접속)

### 프론트 배포 (GitHub Pages)
- `readmelab.github.io` 레포에 push하면 **자동 배포**
- 1~2분 후 `readmelab.github.io/howangdang/v2/` 반영
- 별도 재배포 명령 불필요

## 비상 복구

### 실수로 코드 망가뜨렸을 때
```
git status              # 현재 상태 확인
git diff                # 뭐가 바뀌었는지 보기
git checkout [파일명]    # 특정 파일만 되돌리기
git checkout .          # 모든 변경 되돌리기 (⚠️ 저장 안 된 작업 날림)
```

### 최근 커밋 되돌리기
```
git log                 # 커밋 히스토리 보기
git revert [커밋ID]      # 특정 커밋 취소 (새 커밋으로)
```

**⚠️ `git reset --hard`는 복구 불가능하니 절대 쓰지 말 것**

## 작업 종료 시 루틴

사장님이 **"오늘 작업 마무리할게"** 라고 말하면 Claude는:

1. **완료된 작업 요약:**
   - 어떤 파일이 변경됐는지
   - 커밋이 몇 개 있는지
   - push 완료 여부

2. **CLAUDE.md 진행상황 업데이트:**
   - 완료 항목 → `### 완료` 섹션에 추가
   - 다음 작업 → `### 다음 작업` 섹션 갱신
   - 새 결정사항 → 관련 rules/*.md 파일에 반영

3. **미완료 작업 표시:**
   - push 안 한 커밋이 있으면 알림
   - 재배포 필요한지 확인

4. **다음 세션 가이드:**
   - "다음에 이어서 할 작업: ..."
   - "주의할 점: ..."

## 다중 기기 작업 (사무실 ↔ 집)

### 사무실에서 작업 후
1. 작업 완료 → 커밋 → push
2. VS Code 닫아도 됨

### 집에서 이어서
1. VS Code 열기
2. Claude Code 자동으로 `git pull` 실행
3. 사무실 작업분 자동 반영
4. CLAUDE.md의 "현재 진행 상황" 확인
5. 이어서 작업

**핵심:** 작업 마칠 때마다 **반드시 push**. 안 그러면 다른 기기에서 최신 상태 못 받음.
