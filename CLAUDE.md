## 🚨 0순위 절대 원칙 (모든 규칙 위에 있음)

### 고객 만족 최우선

호왕당의 모든 콘텐츠는 **고객의 실제 불안·걱정·궁금함을 해결**하는 것이 목적이다.

### 작업 기준

1. **"고객이 이 내용으로 만족하는가?"** — 모든 결정의 기준
2. 특히 **직업 관련 상품(직업패/직업처방)** 은 직업군별 실제 불안을 정확히 겨냥해야 함
   - 직장인: 계속 다녀도 되나, 이직해야 하나
   - 사업: 계속 해야 하나, 접어야 하나
   - 프리랜서: 언제까지 이렇게 살 수 있나
   - 주부: 다시 일할 수 있나
   - 취준생: 뭐가 맞는 길인가
   - 학생: 전공·진로 맞게 선택했나

### 파이프라인 업셀링 구조

직업패(무료) → 직업처방(₩9,900) → 재물풀이 Ch.6 → 본풀이 Ch.4

**같은 jobPaeKey 판정이 4단계에서 점점 깊어지는 구조**
- 직업패: 한 줄 훅 (체질 이름)
- 직업처방: 7챕터 진단서 (체질의 정체·원인·처방)
- 재물풀이 Ch.6: "이 체질이 돈을 만드는 방식" (재물 관점 심화)
- 본풀이 Ch.4: "인생 전체에서 이 체질의 의미" (서사 확장)

### 실서비스 안전 원칙

호왕당 V2는 실제 고객이 결제하고 사용 중이다.
수정은 하되, **배포 전 충분한 다각도 테스트**로 파이프라인 안전성을 보장한다.

### 배포 전 필수 테스트 (6레벨)

1. **개별 체질 텍스트 품질** — 5가지 jobPaeKey 각각 검증
2. **파이프라인 전체 흐름** — 직업패→직업처방→재물풀이→본풀이 연결 검증
3. **업셀링 심리 효과성** — 각 단계에서 다음 단계로 유도되는지
4. **시뮬레이션 분포 검증** — 30/25/15/15/15 목표 분포 맞는지
5. **엣지 케이스** — 시간 모름, 경계선 사주 등
6. **프론트 통합** — 레이아웃·결제흐름 문제없는지

### 절대 하지 않는 것

- 테스트 없이 배포
- 단일 체질만 확인하고 판단
- "괜찮겠지" 식 추측
- 사장님 확인 없이 실서버 반영
- 기존 작동 코드의 함수 시그니처·리턴값 변경
- 기존 Firestore 컬렉션 구조 변경

#### 5체질 대비 통일 원칙
모든 체질의 Ch1 base 말미에 5체질 대비 문장 포함:
- 맹수 = 잡식 맹수 (일반)
- 거북이 = wolgeup (월급)
- 여우 = jangsa (장사)
- 부엉이 = jeonmun (전문가)
- 흑표범 = dokgo (독고다이)
- 용 = hanbang (한방)

각 체질별로 자기 체질만 마지막 문장에 배치 + "네가 [동물]이다" 확정.

예 (wolgeup):
> 맹수는 처음부터 혼자 먹는다. 여우는 머리로 먹고, 부엉이는 눈으로 먹는다.
> 흑표범은 혼자의 길로 먹고, 용은 때를 기다려 하늘로 솟는다.
> 거북이는 100년을 간다. 네가 거북이다.

#### Bold 처리 규칙 (B1안 - 5체질 통일)

모든 체질의 직업처방 블록에 마크다운 `**텍스트**` 문법으로 핵심 문장 강조.

**블록 타입별 적용:**
- Ch1 base: 블록당 bold 1개 (체질 정체성 문장)
- Ch1 tail (20개): 블록당 bold 1개 (간지별 핵심 특질 문장)
- Ch2: 블록당 bold 1개 (구조 진단 핵심 문장)
- Ch3: 블록당 bold 1개 (나이대별 행동 원칙 문장)
- Ch4: 블록당 bold 1개 (심리 구조 진단 문장)
- Ch5 (JSON 60블록): bold 없음 (message 축약형)
- Ch6: 블록당 bold 1개 (시기별 처방 핵심 문장)
- Ch7: 블록당 bold 2개 (정체성 문장 + CTA 유도 문장) — 업셀 훅 예외

**선정 원칙:**
- 각 블록에서 "한 줄만 기억해도 이 블록의 처방을 이해할 수 있는" 문장
- 선언형/규정형 문장 우선 ("~다" / "~해라" 끝)
- 비유·설명 문장보다 핵심 지침 문장 우선
- 문장 길이 이상적 20~40자
- 블록이 여러 문단이면 마지막 문단 또는 핵심 결론 문단에 배치 (첫 문단 도입부 금지)
- **예외:** Ch1 base 체질 정체성 앵커는 길이 규정 완화 (3~10자 허용) — 짧을수록 기억 강도 증가 ("네가 여우다" "네가 거북이다" 등)
- **Ch1 tail 전용:** bold는 간지별 자연물 심상(나무/풀/태양/촛불/산/논밭/도끼/보석/강/이슬)을 사용. 체질 앵커 키워드("여우"·"거북이" 등) 사용 금지 — Ch1 base 앵커 독점 보호
- **도입부 정의:** "첫 문단 도입부 금지"의 "도입부"는 체질/간지 호칭 문장에 한정. 호칭 문장이 1번째인 경우 본론 첫 문장(2번째 문장)부터 bold 배치 허용

**근거:** 파이프라인 일관성 유지 (직업처방 → 재물풀이 → 본풀이) + 5체질 통일 UX.

#### 블록 자가 검수 체크리스트 (매 블록 작성 직후 필수)

모든 블록 초안 작성 후, 사장님에게 제시하기 전에 스스로 다음 8개 항목을 확인한다.
하나라도 걸리면 재작성 후 제시.

1. **체질 간 문구 중복** — 다른 체질(wolgeup/jangsa/jeonmun/dokgo/hanbang) 동일 챕터와 문구 겹치는가? (grep 또는 Read로 대조)
2. **챕터 간 문구 중복** — 같은 체질 내 Ch2~Ch7 이미 확정된 표현과 겹치는가? 특히 Ch3↔Ch6, Ch2↔Ch7 연결 지점 주의
3. **블록 내 어휘 반복** — 동일 어휘/구문이 한 블록 안에서 2회 이상 등장하는가?
4. **사주 용어 노출** — 사주 전문 용어(식상생재, 재다신약, 비겁, 관성 등)가 Ch2 외 챕터에 직접 노출되는가? Ch2 외에서는 처방서 톤으로 재가공 필수
5. **이질 심상 돌출** — 해당 체질 앵커 외 새 동물/자연물 심상이 등장하는가? (예: jangsa에 "숲" 갑툭튀)
6. **상품 폄하 뉘앙스** — 파이프라인 상품 간 폄하 표현 있는가? "직업처방만으론 모른다" 같은 표현 금지. 층위 확장 톤으로
7. **화법 일관성** — 행동 지시형("~해라/~마라")인가? 수동 예고문("~때다" 단독, "~줄어든다") 금지
8. **일간 중립성** — 10간 공통 블록(Ch6, Ch7 등)에 특정 일간 명칭(계수 이슬, 갑목 나무 등) 직접 노출되는가?

체크리스트 통과한 블록만 사장님에게 제시. 통과 못한 항목 있으면 자체 수정 후 제시.
