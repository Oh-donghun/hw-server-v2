## 🚨 0순위 절대 원칙 (모든 규칙 위에 있음)

### 고객 만족 최우선

호왕당의 모든 콘텐츠는 **고객의 실제 불안·걱정·궁금함을 해결**하는 것이 목적이다.

### 작업 기준

1. **"고객이 이 내용으로 만족하는가?"** — 모든 결정의 기준
2. 특히 **직업 관련 상품(직업패/직업처방)** 은 직업군별 실제 불안을 정확히 겨냥해야 함
   - 직장인: 계속 다녀도 되나, 이직해야 하나
   - 사업: 계속 해야 하나, 접어야 하나
   - 프리랜서: 언제까지 이렇게 살 수 있나
   - 주부: 다시 일할 수 있나
   - 취준생: 뭐가 맞는 길인가
   - 학생: 전공·진로 맞게 선택했나

### 파이프라인 업셀링 구조

직업패(무료) → 직업처방(₩9,900) → 재물풀이 Ch.6 → 본풀이 Ch.4

**같은 jobPaeKey 판정이 4단계에서 점점 깊어지는 구조**
- 직업패: 한 줄 훅 (체질 이름)
- 직업처방: 7챕터 진단서 (체질의 정체·원인·처방)
- 재물풀이 Ch.6: "이 체질이 돈을 만드는 방식" (재물 관점 심화)
- 본풀이 Ch.4: "인생 전체에서 이 체질의 의미" (서사 확장)

### 실서비스 안전 원칙

호왕당 V2는 실제 고객이 결제하고 사용 중이다.
수정은 하되, **배포 전 충분한 다각도 테스트**로 파이프라인 안전성을 보장한다.

### 배포 전 필수 테스트 (6레벨)

1. **개별 체질 텍스트 품질** — 5가지 jobPaeKey 각각 검증
2. **파이프라인 전체 흐름** — 직업패→직업처방→재물풀이→본풀이 연결 검증
3. **업셀링 심리 효과성** — 각 단계에서 다음 단계로 유도되는지
4. **시뮬레이션 분포 검증** — 30/25/15/15/15 목표 분포 맞는지
5. **엣지 케이스** — 시간 모름, 경계선 사주 등
6. **프론트 통합** — 레이아웃·결제흐름 문제없는지

### 절대 하지 않는 것

- 테스트 없이 배포
- 단일 체질만 확인하고 판단
- "괜찮겠지" 식 추측
- 사장님 확인 없이 실서버 반영
- 기존 작동 코드의 함수 시그니처·리턴값 변경
- 기존 Firestore 컬렉션 구조 변경

## 응답 원칙 (모든 작업에 공통)

1. **사장님은 코딩 문외한이다.** 전문 용어보다 비유로 설명한다.
2. **한 번에 한 질문만.** A/B/C 여러 선택지 동시 제시 금지.
3. **추천은 묻지 않으면 하지 않는다.**
4. **중요한 설계 결정은 반드시 확인 후 진행.** 임의 구현 금지.
5. **대화 끝에 진행상황 요약 제공.**
6. **토큰 절약 최우선.**
7. **모든 대화는 한국어로.**

## 진행상황 업데이트 규칙

사장님이 **"진행상황 업데이트해줘"** 라고 말하면 Claude는:
1. 완료된 항목 → `### 완료` 섹션에 추가
2. 다음 작업 → `### 다음 작업` 섹션 갱신
3. 새 결정사항 → 해당 상세 규칙 파일(.claude/rules/)에 반영
4. 변경사항 요약 보고
